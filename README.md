# Three spouses – logic programming

## Постановка задачі
Задача «Три подружжя» — класична логічна задача на переправу через річку.
Є три подружні пари, які повинні переправитися з лівого берега на правий за допомогою човна.

Обмеження задачі:
- Човен може перевозити не більше двох осіб.
- Жодна жінка не може залишатися на березі з чужим чоловіком без присутності свого чоловіка.
- Потрібно знайти коректну послідовність переправ, яка задовольняє всі обмеження.

---

## Підхід до розв’язання
Задача реалізується мовою **Prolog** з використанням:
- представлення стану (лівий берег, правий берег, положення човна);
- правил перевірки коректності станів;
- пошуку в просторі станів (BFS / DFS);
- запобігання повторенню вже відвіданих станів.

## Порівняння з реалізацією на Haskell:
### Prolog
- Описує задачу **декларативно**, через факти та правила.
- Обмеження формулюються явно у вигляді логічних предикатів.
- Пошук розв’язку природно реалізується через механізм логічного виведення (BFS / DFS).
- Добре підходить для задач з умовами та обмеженнями.

### Haskell
- Розв’язання будується через **чисті функції** та явне керування станами.
- Потрібно явно описувати переходи між станами та механізм пошуку.
- Контроль циклів і повторних станів реалізується через структури даних (списки, множини).
- Підхід більш алгоритмічний за стилем, ніж у Prolog.

### Загальне порівняння та висновок
- У Prolog задача формулюється ближче до **умови задачі**, без явного керування алгоритмом.
- У Haskell розробник має більший контроль над виконанням, але за рахунок складнішого коду.
- Для задач логічного пошуку з обмеженнями Prolog є більш природним і компактним.
